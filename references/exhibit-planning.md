# Exhibit planning

Every slide must have a page brief before production.

## Page brief template

| Field | Description |
|---|---|
| Key question | The business question this page answers |
| Action title | The conclusion to be placed at the top of the slide |
| Evidence | Facts, analysis, client data or explicit assumptions supporting the title |
| Exhibit type | Chart, table, 2x2, roadmap, architecture, operating model, etc. |
| Data source | Source, calculation basis or `team assumption` |
| Implication | What this means for the audience's decision |
| Caveat | Limitations, assumptions or uncertainties |

## Quality bar

- No evidence, no page.
- The exhibit must prove the action title, not merely decorate the page.
- If a data point is illustrative, label it clearly.
- If the evidence is complex, move supporting detail into appendix.

## Example

| Field | Example |
|---|---|
| Key question | Which AI use cases should scale first? |
| Action title | Internal knowledge workflows provide the best first-wave balance of value and readiness |
| Evidence | Use-case scoring across value, data availability, risk and reuse |
| Exhibit type | 2x2 with insight rail |
| Data source | Client workshop scoring; team assumption where missing |
| Implication | Start with internal copilots before external customer chatbots |
| Caveat | Scores require validation with business owners |
