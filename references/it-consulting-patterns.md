# IT consulting patterns

## Cloud migration

Use 6R language where appropriate: rehost, replatform, refactor, repurchase, retire, retain. Common exhibits: application portfolio 2x2, landing-zone architecture, migration wave roadmap, TCO waterfall and operating model.

## AI transformation

Core modules: use-case prioritization, data readiness, model/platform architecture, AI governance, evaluation, adoption and value tracking. Common exhibits: value/readiness 2x2, capability maturity map, target AI platform architecture, hub-and-spoke operating model and risk matrix.

## Application modernization

Core modules: technical debt, business criticality, modernization pattern, target architecture and migration waves. Common exhibits: app portfolio 2x2, dependency map, modernization roadmap and business case.

## Data platform

Core modules: data product strategy, lakehouse/data mesh architecture, governance, lineage, quality, access control and analytics operating model. Common exhibits: layered data architecture, capability map and governance RACI.

## Security and compliance

Core modules: zero trust, control mapping, data classification, auditability, incident response and regulatory requirements. Common exhibits: risk matrix, control heatmap, target-state security architecture and remediation roadmap.

## Business case

Use CAPEX/OPEX, run-rate savings, productivity benefits, payback and sensitivity analysis. Always show assumptions and calculation basis.

## Adoption and operating model

Cover CoE, product ownership, training, change management, business champions, governance cadence and benefits tracking. Common exhibits: hub-and-spoke model, RACI, cadence chart and adoption funnel.
